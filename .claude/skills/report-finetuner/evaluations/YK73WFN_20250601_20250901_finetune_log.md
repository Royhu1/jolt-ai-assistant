## Unfixable defects (routed out)

Two defect classes appeared that **cannot** be corrected with the current
finetune API and should be routed to higher-layer owners:

### Class A — Charge-session fragmentation (4 occurrences)

SRF telematics data gaps cause the charger patcher to terminate a charge
session early, producing a `Charge_X → Stop → Charge_X` sandwich that is
really one continuous session. Raw `battery_pack_charging_status` column
confirms `CHARGING_AC/DC` throughout including the interior Stop.

Merging these via `MergeOp(new_type="DC Home/AC Home")` technically
reconstructs the right leg type, but per the well-known gotcha (see
agent prompt §陷阱 #2 and `YK73WFN.md`), the merged row's
`Energy Change (kWh)` column is recomputed from SOC × effective_capacity
**not** integrated from the raw cumulative `total_electric_energy_used`
column. For charge rows this would replace the accurate charger-patcher
kWh (`Energy Output from Charger`, `Peak Charging`, `Average Charging`)
with a SOC-derived estimate — a regression in data quality.

**Recommendation**: Route to `jolt-report-dev` as a request to make the
charger patcher more tolerant of `electricCharging=True` gaps, or to
extend `finetune.py` with a "merge charge without EC overwrite" flag.

Affected rows:
- Row 238+239 (2025-07-11 23:16 → 2025-07-12 02:05)
- Row 284+285+286 (2025-07-15 23:38 → 2025-07-16 00:53)
- Row 418+419+420 (2025-07-26 23:20 → 2025-07-27 00:41)
- Row 529+530+531 (2025-08-09 21:00 → 2025-08-10 02:48)
- Row 597+598+599 (2025-08-17 20:49 → 2025-08-18 01:51)
- Row 667+668+669 (2025-08-24 22:07 → 2025-08-25 04:17)

### Class B — Trip/charge hidden inside a long Stop (3 occurrences)

Sparse telematics leaves the trip/charge detector blind to short activity
windows buried in multi-hour Stops. Raw CSV confirms either a real drive
(>50 km at >80 km/h) or a short DC charge happened inside the Stop.

Since `finetune.py` exposes only Merge (combine rows), Split (one row
→ two rows), and Delete (remove one row), **there is no operation that
inserts a new leg** into an existing Stop. The only workaround
(Split the Stop, then "merge" with phantom rows that don't exist) is
not supported.

**Recommendation**: Route to `jolt-report-dev` as a request to add an
`InsertOp(before_row, new_leg)` operation, or tune the segmenter's
zero-speed blackout tolerance so it re-opens a trip when activity
resumes mid-Stop.

Affected rows:
- Row 411 (2025-07-25 23:19 → 2025-07-26 15:39) — missed DC mini-charge
- Row 449 (2025-07-29 04:51 → 2025-07-29 06:10) — missed 70-km second trip
- Row 710 (2025-08-31 21:26 → 2025-09-01 17:00) — missed charge
