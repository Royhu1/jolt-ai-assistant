## Edge cases / observations

1. **Telematics data corruption 2025-01-23 to 2025-01-24 morning**. SOC samples
   briefly crash to 9% and recover to 82% several times while vehicle is stationary
   and `total_electric_energy_used` is unchanged. The algorithm correctly
   identifies them as alternating "In House" discharges and "Charge Home" events
   (rows 84-96). **This is a telematics signal quality problem, not a
   segmentation error.** Fixing it would require discarding / imputing corrupt
   SOC samples, which is upstream of `jolt_toolkit` segmentation and belongs
   either to SRF data providers or a future data-QC layer. Out of scope for
   `report-finetuner`; flagging for potential follow-up by `jolt-report-dev`.

2. **Winter EP depression**. Rows in December-February show consistently higher
   kWh/km than summer reference period — this is expected cold-battery physics,
   not an artefact.

3. **Short low-load In House legs** (2024-12-22 r57, 2025-02-14 r149/151) produce
   very high EP values (3–13 kWh/km). These are mathematically correct but
   quantised SOC (1% = ~4 kWh at C=400 kWh) amplifies them. Not a finetune issue.

4. **No recurring pattern** would benefit from a param-tuner re-run. The existing
   parameters handle the winter period cleanly.
