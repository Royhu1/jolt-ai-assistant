# N88GNW — Renault Trucks D Wide Z.E.

## 车辆特征

- 品牌/型号: Renault Trucks D Wide Z.E.
- 标称容量: 540 kWh
- 运营模式: 多点配送（城市/近郊），典型每日 10-14 个停靠点
- 数据特征: 遥测 SOC + 遥测速度 (`wheel_based_speed`) + Logger Mass；AC/DC 充电功率列不稳定（经常为零或累计值）
- 数据范围: 2024-10-11 至 2025-12-01（421 天，5 个季度报告）

## 管线与参数

- Pipeline: `renault_speed` (speed-based 分段)
- 所有参数均为默认值，无需特殊调整

| 参数 | 值 | 原因 |
|------|-----|------|
| `speed_threshold_kmh` | 1.0 | 默认；正确区分静止/移动 |
| `min_stop_duration_min` | 5.0 | 默认；适合多点配送停靠模式 |
| `min_trip_duration_min` | 2.0 | 默认；捕捉短距离调车 |
| `min_soc_drop` | 1.0 | 默认；540 kWh 下 1% = 5.4 kWh，合理 |
| `min_energy_kwh` | 1.0 | 默认；过滤微动 |

## 关键发现

### 1. Renault AC/DC 充电数据不可靠
Renault D Wide Z.E. 的遥测 AC/DC 充电功率列行为不一致：
- 部分天数完全为零（即使 SOC 明显上升）
- 部分天数显示累计电量计值（~100-400 kWh 阶跃），而非瞬时功率
- 充电检测正确回退到 SOC-rise 启发式方法

### 2. 高质量速度分段
speed-based 管线在 540 kWh 大容量车辆上表现出色：
- 典型工作日产生 1-14 个放电段，与速度 trace 完美对齐
- 多点配送模式下 `min_stop=5.0` 分钟正确区分「停靠」vs「行驶中减速」
- EP 值 0.8-2.5 kWh/km 范围合理（~26t GVW 级别）

### 3. 质量数据噪声大但可用
- 原始遥测质量范围 10,000-35,000 kg（含噪声/跳变）
- 分段均值质量稳定在 14-25 t（合理的 laden/unladen 范围）
- 偶尔可见装货事件（质量从 ~10 t 跳升至 ~30 t）

### 4. 跨日边界充电遗漏（已确认为架构限制）
- 约 33/421 天（7.8%）出现此问题
- 夜间充电在 00:00 之前完成 -> 归入前一天
- 或充电在 00:00 之后开始 -> 当天只有放电
- 这是 per-day 分段窗口的固有限制，不可通过参数调整解决

## 经验教训（适用于类似 Renault 车辆）

1. **Renault 默认参数即可** — 无需像 Mercedes 那样调整 `min_soc_drop` 或切换速度列
2. **不要依赖 AC/DC 充电数据** — Renault 遥测的 AC/DC 列不可靠，充电检测应以 SOC-rise 为主
3. **多点配送模式下 `min_stop=5.0` 适用** — 比长途运输（可能需要更大 min_stop）更适合高频停靠
4. **质量数据噪声需要在分析时考虑** — 使用分段均值而非原始时序值
5. **大容量车辆（540 kWh）的 `min_soc_drop=1.0` 对应 ~5.4 kWh** — 足够过滤噪声而不遗漏有效行程
