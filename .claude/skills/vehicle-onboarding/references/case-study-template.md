# Post-onboarding case study — template for `references/{reg}.md`

Open on demand at Phase 6 (see `static/core/workflow.md`). Per-vehicle onboarding
experience accumulates in these case studies — this skill's equivalent of an
`evaluations/` log (skill-design principle 7).

After successful onboarding, save `references/{reg}.md` with:

1. Vehicle specs (make, model, capacity, data characteristics)
2. Telematics column mapping (which standard columns exist, any non-standard names)
3. Algorithm choices made and why
4. Initial parameter values and any tuning done
5. Data quirks discovered (SOC precision, missing channels, etc.)

These case studies help future onboarding of similar vehicles.

Existing examples: [T88RNW.md](T88RNW.md) (Renault E-Tech D Wide, rigid EV),
[TA70WTL.md](TA70WTL.md) (Renault E-Tech T, artic EV),
[YT21EFD.md](YT21EFD.md) (Scania P410, diesel / SRF-Logger path).
