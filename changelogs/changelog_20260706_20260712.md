**Part 1 — charger fusion: root causes diagnosed (verified against live SRF queries) + fixed.**
Four compounding causes: (a) `ChargerPatcher.patch_file` only matched leg types containing
`AC`/`DC`, silently skipping the generic `Charge Home` / `Charge Away` legs — which are ALL
of EX74JXW/JXY, LN25NKE, YN25RSY, YN75NMA's charging legs (EX74JXY: 89 SRF transactions, 0
links ever); (b) the 2.2.7 database was produced by the SRF-free recompute, which preserved
charger links verbatim from 2.2.6 and did NOT carry `raw_charger/` forward — the dashboard
raw base and the monitor digest read that CSV, so T88RNW/N88GNW charger visibility vanished;
(c) `_save_charger_data` overwrote `charger_transactions.csv` with only the current window's
transactions (T88RNW's 2.2.6 CSV held just 3 rows); (d) fusion only ever ran at generation
time — transactions arriving late in SRF were never backfilled. T88RNW/N88GNW xlsx links
were actually ~consistent with what SRF associates to them today (93/102, 128/154).
**Fix** (jolt-toolkit-dev agent, branch `fix/charger-fusion`, commit `d13b0a4`): leg filter
extended to `Charge`; `_find_charger_matches` collects ALL overlapping windows (±4 min) and
sums their energies (dual-gun proven: EX74JXY 157.906+212.358=370.264 kWh into one leg),
links the earliest URI; shared `merge_save_charger_transactions` helper (concat → dedup by
uri → sort by start_time) used by both the generator and the new backfill CLI
`python -m jolt_toolkit.report_generator.charger_patcher <file|veh-dir|version-dir>
[--persist-raw]` (idempotent — only fills empty link cells; skips diesel). Verified on tmp
copies: EX74JXY 0→40 links (the other ~37 tx fall in telematics gaps with no overlapping
charge leg — correctly unmatched), T88RNW 93→94 + raw CSV accumulated to 102 unique tx;
second run adds 0 (idempotency + dedup confirmed).

**Part 2 — capacity×ΔSOC fallback: design validated on data, implementation delegated.**
Per-leg stats from all 2.2.7 reports + visual validation-figure forensics established the
failure mode as **stale counter anchors** (endpoint snapshots far from trip boundaries →
paired under/over-attribution; exemplar T88RNW 2025-09-29: 38.55 km leg recorded −0.47 kWh,
next leg swallowed the 73 kWh backlog at implied capacity 375 vs median 214 kWh; YK73WFN /
EV73SAL / N88GNW show 70–90 km legs at implied capacity 840–1070 kWh, EP 2.4–4.9). SOC
channels on these vehicles are densely sampled and reliable. Dry-run of the mechanism:
YK73WFN rewrites 3.9% of trips, EV73SAL 6.1%, N88GNW 2.7%, T88RNW 0.8%; median EP of
rewritten legs 1.8–1.9 → 1.2–1.3 kWh/km. **Design**: inside `_correct_effective_capacity`
step 2, for ±1σ-outlier counter-sourced rows of opted-in vehicles with |ΔSOC| ≥ 10% and
